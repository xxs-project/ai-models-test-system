## 简介

大模型测试平台是一个统一的大模型性能测试管理系统，提供设备管理、测试执行、结果分析等完整功能。

### 主要功能

- **设备管理**: 添加、编辑、删除测试设备，实时监控设备状态
- **测试管理**: 创建、执行、监控测试任务
- **结果呈现**: 性能对比、趋势分析、报告生成
- **系统设置**: 配置监控参数、通知设置等

## 快速入门

### 首次登录

1. 打开浏览器访问 http://localhost:5175
2. 默认进入仪表板页面
3. 开始添加您的第一台测试设备

### 添加设备

1. 点击左侧菜单 **设备管理**
2. 点击右上角 **添加设备**
3. 填写设备信息：
   - IP 地址
   - 端口（默认 22）
   - 用户名
   - SSH 密码
   - 备注（可选）
4. 点击 **提交** 添加设备

## 设备管理

### 添加新设备

1. 进入 **设备管理** 页面
2. 点击 **添加设备** 按钮
3. 填写设备连接信息
4. 点击 **提交**

### 设备列表操作

| 操作 | 说明 |
|------|------|
| 刷新 | 手动刷新设备状态 |
| 编辑 | 修改设备连接信息 |
| 删除 | 删除设备（不可恢复） |

### 筛选设备

- **按状态筛选**: 在线 / 离线 / 未知
- **按架构筛选**: x86_64 / aarch64
- **搜索**: 支持按 IP、用户名搜索

### 查看设备详情

点击设备行可查看详细信息：
- 操作系统信息
- 加速卡状态
- 历史任务记录

## 测试管理

### 创建测试任务

1. 进入 **测试管理** 页面
2. 点击 **创建任务** 按钮
3. 填写任务配置：

#### 基本信息
- **任务名称**: 描述性名称
- **优先级**: 低 / 中 / 高
- **任务描述**: 详细说明

#### 设备配置
- **选择设备**: 从已添加的设备中选择
- **加速卡数量**: 使用的加速卡数量

#### 模型配置
- **模型名称**: 如 Qwen-14B
- **模型路径**: 模型文件路径
- **推理框架**: MindIE / VLLM
- **框架版本**: 框架版本号

#### 测试参数
- **上下文长度**: 如 1024, 2048, 4096
- **并发数**: 如 1, 8, 16, 32, 64
- **测试轮次**: 每组参数测试次数

4. 点击 **创建** 提交任务

### 执行测试

1. 在任务列表中找到目标任务
2. 点击 **执行** 按钮
3. 任务状态变为 "执行中"
4. 可查看实时进度

### 监控任务

任务执行时可查看：
- 执行进度百分比
- 当前执行阶段
- 预计剩余时间

### 取消任务

任务执行中可点击 **取消** 终止执行。

### 任务状态说明

| 状态 | 说明 |
|------|------|
| 待执行 | 任务已创建，等待执行 |
| 队列中 | 任务排队中 |
| 准备中 | 环境准备中 |
| 执行中 | 测试进行中 |
| 已完成 | 测试正常完成 |
| 失败 | 测试执行失败 |
| 已取消 | 用户手动取消 |
| 超时 | 执行超时 |

## 结果呈现

### 基准测试列表

查看所有基准测试数据，支持：
- 搜索筛选
- 按框架筛选
- 选择对比

### 性能对比

1. 在基准测试列表中勾选两个测试
2. 点击 **对比** 按钮
3. 查看对比结果：

#### 对比维度
- **TPS**: 每秒 tokens 数
- **TTFT**: 首 token 时间
- **TPOT**: 每 token 输出时间
- **并发性能**: 不同并发数下的表现

#### 性能趋势
- 可视化图表展示
- 支持多版本对比

### 导出报告

1. 在对比页面输入总结
2. 点击 **保存报告**
3. 可导出为 HTML 格式

## 常见问题

### Q1: 设备添加失败

可能原因：
- SSH 连接信息错误
- 设备网络不可达
- SSH 端口未开放

解决方法：
1. 检查 IP、端口、用户名、密码
2. 确认设备网络可达
3. 检查防火墙设置

### Q2: 测试执行缓慢

可能原因：
- 设备资源不足
- 并发数过高
- 模型过大

解决方法：
1. 减少并发数
2. 选择资源更充裕的设备
3. 使用模型量化版本

### Q3: 数据丢失

预防措施：
1. 定期导出重要数据
2. 启用自动备份
3. 及时保存报告
